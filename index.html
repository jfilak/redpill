SCM in ABAP

  • gCTS
  • abapGit

gCTS -Git enabled CTS

  • transports to commits
  • random set of objects
  • support for almost all object types
  • machine format serialization
  • kinda fast operations
  • switching branches works nice
  • fails to deal with rewritten history
  • remote API available

abapGit

  • ABAP package to git repo mapping (git native)
  • support only for subset of objects types
  • human readable format
  • switching branches somehow fragile
  • does not handle big packages very well
  • mainly focused on human interaction

gCTS vs. abapGit

  • integrate vs. innovate
  • transparent vs. explicit
  • complicated vs. simple
  • comprehensive vs. minimalistic
  • enterprise support vs. community support
  • complete livecycle vs. mainly backup
  • limited availability vs. works on >7.04

Branching variants

  • standard git options
  • does not depend on tool
  • feature branch or cherry-picking

Private development systems

  • creating a snapshot is simple
  • dealing with hostname changes is not

Continuous Integration

  • Jenkins, GitHub Actions, Travis, MS Azure, ...
  • easier to start a docker container than provision a new VM
